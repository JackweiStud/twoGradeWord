# 💻 AI 代码修改提示词模板

给 AI Code Assistant 编写代码修改提示词时，最重要的是提供**清晰的上下文、完整的待修改代码**以及**明确的修改目标**。

以下是一份结构化的、针对代码修改的提示词模板：

---

## 📌 一、背景说明 (Context and Rationale)

| 提示词内容 | 说明/示例 |
| :--- | :--- |
| **【项目名称/模块】** | [例如：TwoGradeWord Web UI 项目 / 词库加载模块] |
| **【技术栈】** | [例如：Vue 3 + Composition API, JavaScript, Vite 构建] |
| **【当前功能】** | [例如：该函数目前用于从本地资源 `/words/wordsSource.json` 异步加载词库数据。] |
| **【需修改的原因】** | [例如：部署后在 Vercel 环境中，由于资源路径或 CORS 问题，`fetch` 抛出异常，需要增加健壮的错误处理和更清晰的错误提示。] |
| **【运行环境】** | [例如：已部署在 Vercel，本地 `npm run dev` 正常。] |

---

## 📋 二、待修改代码内容 (Code Snippet)

请提供完整的、需要修改的函数或代码块，以便 AI 准确理解上下文。

```javascript
// [请将您的原始代码粘贴在这里]

async loadWordsData() {
    try {
        const response = await fetch('/words/wordsSource.json')
        // ... 后续处理数据的逻辑 ...
    } catch (error) {
        // ... 现有的错误处理逻辑 ...
    }
}
```

---

## 🎯 三、修改目标与具体要求 (Modification Goal)

清晰地告诉 AI 您希望代码修改成什么样子。

| 提示词内容 | 具体要求/示例 |
| :--- | :--- |
| **【核心目标】** | [**确保在 Vercel 部署环境下的数据加载成功**，并处理网络错误。] |
| **【修改要求 1：错误处理】** | [**增强 `try...catch` 块**：如果 `fetch` 失败，不仅要捕获错误，还需要在控制台打印详细的 `error.message`，并在前端界面显示"词库加载失败，请检查网络或资源路径"的提示。] |
| **【修改要求 2：路径修正】** | [如果可能，请**优化 `fetch` 路径的写法**，使其在开发和部署环境都稳定（例如，使用 `import.meta.env.BASE_URL`）。] |
| **【输出格式要求】** | [**只输出修改后的完整 `loadWordsData` 函数代码块**，并简要解释修改点。] |

---

>
> ```javascript
> // ... (粘贴您的原始代码)
> ```
>
> **修改目标：**
>
> 1. 确保 `fetch` 成功时数据正常处理。

> 3. **请只输出修改后的完整代码块**。"

---

## 🔑 关键要点总结

1. **上下文清晰**：明确说明项目背景、技术栈和当前功能
2. **代码完整**：提供完整的待修改代码片段，不要省略关键部分
3. **目标明确**：清晰列出修改目标和具体要求
4. **格式规范**：使用结构化的表格或列表，便于 AI 理解
5. **输出要求**：明确说明期望的输出格式（完整代码块、解释说明等）

---

## 💡 使用建议

- **复杂修改**：对于复杂的修改需求，建议先拆解成多个小步骤，逐步描述
- **边界情况**：明确说明需要处理的边界情况和异常场景
- **兼容性**：如果涉及多环境兼容，明确说明各个环境的差异
- **测试要求**：如果对修改后的代码有测试要求，请在提示词中说明
